# دليل نشر نظام الخزنة على Render
# Treasury Management System - Render Deployment Guide

## المتطلبات
- حساب على موقع Render.com
- مشروع مرفوع على GitHub

## خطوات النشر

### 1. رفع المشروع على GitHub
```bash
git add .
git commit -m "Prepare for Render deployment"
git push origin main
```

### 2. إنشاء خدمة جديدة على Render

1. اذهب إلى [render.com](https://render.com)
2. سجل الدخول أو أنشئ حساب جديد
3. اضغط على "New +" ثم اختر "Web Service"
4. اختر "Build and deploy from a Git repository"
5. اربط حساب GitHub الخاص بك
6. اختر المستودع الذي يحتوي على المشروع

### 3. إعدادات الخدمة

**اسم الخدمة:** `treasury-management-system`

**البيئة:** `Python 3`

**الفرع:** `main`

**أمر البناء (Build Command):**
```bash
pip install -r requirements.txt
```

**أمر التشغيل (Start Command):**
```bash
python app.py
```

### 4. متغيرات البيئة (Environment Variables)

لا تحتاج إلى متغيرات بيئة إضافية، التطبيق سيعمل بالضبط كما هو.

### 5. بيانات تسجيل الدخول الافتراضية

- **اسم المستخدم:** `admin`
- **كلمة المرور:** `password`

### 6. النشر

1. اضغط على "Create Web Service"
2. انتظر حتى يكتمل البناء والنشر (5-10 دقائق)
3. ستحصل على رابط للتطبيق مثل: `https://treasury-management-system.onrender.com`

## ملاحظات مهمة

- التطبيق يستخدم SQLite كقاعدة بيانات محلية
- البيانات ستُحفظ في قاعدة البيانات على الخادم
- التطبيق سيعيد تشغيل قاعدة البيانات في كل مرة يتم فيها إعادة النشر
- للاستخدام الإنتاجي، يُنصح باستخدام قاعدة بيانات خارجية مثل PostgreSQL

## استكشاف الأخطاء

إذا واجهت مشاكل:

1. تحقق من سجلات البناء (Build Logs)
2. تحقق من سجلات التشغيل (Runtime Logs)
3. تأكد من أن جميع الملفات موجودة في المستودع
4. تأكد من أن `requirements.txt` يحتوي على جميع المكتبات المطلوبة

# خزينة هرمية - نظام إدارة المالية الهرمي

## الوصف
نظام شامل لإدارة المال والإيرادات والمصروفات بشكل هرمي ومنظم. يتيح للمستخدمين إدارة مواردهم المالية بطريقة مرنة واحترافية مع إمكانية تخصيص الخزائن والمشاريع.

## الميزات الرئيسية

### 🏦 إدارة الخزائن الهرمية
- خزينة رئيسية تحتوي على رصيد إجمالي
- خزائن فرعية للمشاريع أو فئات معينة
- هيكل هرمي مرن وقابل للتوسع
- صفحات تفصيلية لكل خزينة مع إحصائيات شاملة
- نماذج إضافة وتعديل الخزائن

### 📊 إدارة المشاريع
- إنشاء مشاريع جديدة مع خزائن مخصصة
- تتبع الأداء المالي على مستوى المشروع
- حالات مختلفة للمشاريع (نشط، مكتمل، معلق)
- صفحات تفصيلية للمشاريع مع التقارير المرتبطة
- نماذج إضافة وتعديل المشاريع

### 💰 التقارير المالية
- تسجيل الإيرادات والمصروفات
- تقارير تفصيلية لكل خزينة ومشروع
- إحصائيات شاملة ولوحة تحكم
- فلاتر بحث متقدمة
- نماذج إضافة وتعديل التقارير
- ميزات تصدير البيانات (CSV, JSON, TXT)

### 👥 إدارة المستخدمين والأدوار
- نظام أدوار (أدمن، مستخدم)
- صلاحيات مختلفة لكل دور
- إدارة شاملة للمستخدمين
- صفحة الملف الشخصي مع تغيير كلمة المرور
- إحصائيات المستخدمين

### 📈 لوحة التحكم المتقدمة
- إحصائيات شاملة ومؤشرات الأداء
- مخططات بيانية تفاعلية (Bar, Pie, Line)
- التقارير الأخيرة
- نظرة عامة على النظام

### 🔔 نظام الإشعارات
- إشعارات فورية للعمليات المهمة
- أنواع مختلفة من الإشعارات (نجاح، خطأ، تحذير، معلومات)
- إشعارات تختفي تلقائياً
- واجهة مستخدم جذابة

### 📊 المخططات البيانية
- مخططات أعمدة لعرض الأرصدة
- مخططات دائرية لتوزيع الإيرادات والمصروفات
- مخططات خطية للاتجاهات الزمنية
- مكونات قابلة لإعادة الاستخدام

### 📤 ميزات التصدير
- تصدير البيانات بصيغة CSV
- تصدير البيانات بصيغة JSON
- تصدير البيانات بصيغة TXT
- تنسيق البيانات تلقائياً حسب النوع

## التقنيات المستخدمة

### الخلفية (Backend)
- **Node.js** + **Express.js**: خادم API
- **MySQL**: قاعدة البيانات
- **JWT**: المصادقة والأمان
- **bcryptjs**: تشفير كلمات المرور

### الواجهة الأمامية (Frontend)
- **React 19**: مكتبة واجهة المستخدم
- **Vite**: أداة البناء والتطوير
- **Tailwind CSS**: تصميم واجهة المستخدم
- **React Router**: توجيه الصفحات
- **Axios**: طلبات HTTP

## التثبيت والتشغيل

### المتطلبات
- Node.js 16+
- MySQL 8.0+ أو PostgreSQL 12+
- npm أو yarn

### 🚀 التشغيل السريع

#### الطريقة الأولى: استخدام Neon (مستحسن)
```bash
# تشغيل السكريبت التلقائي مع Neon
./quick-start-neon.sh
```

#### الطريقة الثانية: استخدام MySQL
```bash
# تشغيل السكريبت التلقائي مع MySQL
./start.sh
```

#### الطريقة الثانية: التشغيل اليدوي

### 1. إعداد قاعدة البيانات

#### MySQL:
```bash
# الدخول إلى MySQL
mysql -u root -p

# تشغيل ملف إنشاء قاعدة البيانات
source /path/to/hierarchical-vault/database/schema.sql
```

#### PostgreSQL:
```bash
# الدخول إلى PostgreSQL
psql -U postgres

# تشغيل ملف إنشاء قاعدة البيانات
\i /path/to/hierarchical-vault/database/schema-postgresql.sql
```

### 2. إعداد الخادم الخلفي

```bash
cd backend
npm install
cp .env.example .env
# تحديث ملف .env بالقيم المناسبة
npm run dev
```

### 3. إعداد الواجهة الأمامية

```bash
cd frontend
npm install
# تأكد من وجود ملف .env مع VITE_API_URL=http://localhost:5000/api
npm run dev
```

### 🐳 التشغيل باستخدام Docker

```bash
# تشغيل جميع الخدمات
docker-compose up -d

# إيقاف الخدمات
docker-compose down
```

## API Endpoints

### المصادقة
- `POST /api/auth/login` - تسجيل الدخول
- `POST /api/auth/logout` - تسجيل الخروج
- `GET /api/auth/me` - معلومات المستخدم الحالي

### الخزائن
- `GET /api/vaults` - الحصول على جميع الخزائن
- `POST /api/vaults` - إنشاء خزينة جديدة
- `PUT /api/vaults/:id` - تحديث خزينة
- `DELETE /api/vaults/:id` - حذف خزينة

### المشاريع
- `GET /api/projects` - الحصول على جميع المشاريع
- `POST /api/projects` - إنشاء مشروع جديد
- `PUT /api/projects/:id` - تحديث مشروع
- `DELETE /api/projects/:id` - حذف مشروع

### التقارير المالية
- `GET /api/reports` - الحصول على التقارير
- `POST /api/reports` - إنشاء تقرير جديد
- `PUT /api/reports/:id` - تحديث تقرير
- `DELETE /api/reports/:id` - حذف تقرير

# 🚀 النشر بعد نقل ملفات Frontend

## ✅ تم نقل ملفات Frontend إلى الجذر

الآن ملفات Frontend موجودة في الجذر، يمكنك النشر كالتالي:

## 1️⃣ إعدادات Backend Service

**Basic Settings:**
- **Name**: `hierarchical-vault-backend`
- **Environment**: `Node`
- **Root Directory**: `backend`
- **Build Command**: `npm install`
- **Start Command**: `npm start`

**Environment Variables:**
```
NODE_ENV=production
PORT=10000
DB_HOST=YOUR_NEON_HOST_HERE
DB_PORT=5432
DB_NAME=YOUR_NEON_DATABASE_HERE
DB_USER=YOUR_NEON_USERNAME_HERE
DB_PASSWORD=YOUR_NEON_PASSWORD_HERE
DB_SSL=true
JWT_SECRET=your-secret-key-here
JWT_EXPIRES_IN=24h
CORS_ORIGIN=https://hierarchical-vault-frontend.onrender.com
```

## 2️⃣ إعدادات Frontend Service

**Basic Settings:**
- **Name**: `hierarchical-vault-frontend`
- **Environment**: `Static Site`
- **Root Directory**: `(فارغ - الجذر)`
- **Build Command**: `npm install && npm run build`
- **Publish Directory**: `dist`

**Environment Variables:**
```
VITE_API_URL=https://hierarchical-vault-backend.onrender.com/api
```

## 3️⃣ خطوات النشر

### الخطوة 1: نشر Backend
1. اذهب إلى Backend Service
2. اضغط "Manual Deploy" → "Deploy latest commit"
3. انتظر اكتمال البناء

### الخطوة 2: نشر Frontend
1. اذهب إلى Frontend Service
2. اضغط "Manual Deploy" → "Deploy latest commit"
3. انتظر اكتمال البناء

### الخطوة 3: تحديث CORS
1. احصل على Frontend URL من Render
2. اذهب إلى Backend Service → Environment
3. حدث `CORS_ORIGIN` بالـ Frontend URL الجديد
4. اضغط "Save Changes"
5. اضغط "Manual Deploy" مرة أخرى

## 4️⃣ اختبار المشروع

### Backend Test:
- اذهب إلى Backend URL
- يجب أن ترى: `{"message":"Hierarchical Vault API"}`

### Frontend Test:
- اذهب إلى Frontend URL
- يجب أن ترى صفحة تسجيل الدخول

## 🔧 استكشاف الأخطاء

إذا واجهت مشاكل:
1. تحقق من Logs في Render
2. تأكد من صحة متغيرات البيئة
3. تأكد من اتصال قاعدة البيانات

## 📝 ملاحظات مهمة

- ✅ ملفات Frontend الآن في الجذر
- ✅ لا حاجة لـ Root Directory للـ Frontend
- ✅ Build Command: `npm install && npm run build`
- ✅ Publish Directory: `dist`

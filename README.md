# نظام إدارة الخزينة الاحترافي

نظام شامل لإدارة الخزائن والعمليات المالية مبني باستخدام React و Node.js و PostgreSQL.

## 🌟 المميزات الرئيسية

### إدارة الخزائن
- **الخزينة الرئيسية**: إدارة الرصيد الإجمالي للنظام
- **الخزائن الفرعية**: خزائن متخصصة للمصروفات اليومية، الموردين، والمشاريع
- **تتبع الأرصدة**: مراقبة مستمرة لأرصدة جميع الخزائن
- **إدارة الصلاحيات**: تحديد صلاحيات الوصول للخزائن والعمليات

### العمليات المالية
- **الإيداع**: إضافة الأموال إلى الخزائن
- **السحب**: سحب الأموال من الخزائن مع التحقق من الرصيد
- **التحويل**: تحويل الأموال بين الخزائن المختلفة
- **تسوية الحسابات**: ضبط التوازن بين الخزائن

### التقارير والتحليلات
- **التقارير المالية**: تقارير شاملة عن الحركات المالية
- **الرسوم البيانية**: عرض البيانات بصرياً باستخدام Chart.js
- **تحليل الأداء**: تقييم أداء كل خزينة على حدة
- **اتجاهات المعاملات**: تتبع الاتجاهات المالية عبر الوقت

### الأمان والمصادقة
- **نظام تسجيل الدخول**: مصادقة آمنة للمستخدمين
- **إدارة الأدوار**: صلاحيات مختلفة للمديرين والمستخدمين
- **تشفير كلمات المرور**: حماية آمنة للبيانات الحساسة
- **JWT Tokens**: نظام مصادقة متقدم

## 🛠️ التقنيات المستخدمة

### الواجهة الأمامية (Frontend)
- **React 18** مع TypeScript
- **Tailwind CSS** للتصميم
- **React Router** للتنقل
- **Chart.js** للرسوم البيانية
- **Lucide React** للأيقونات
- **Axios** للاتصال بالخادم

### الواجهة الخلفية (Backend)
- **Node.js** مع Express.js
- **PostgreSQL** قاعدة البيانات
- **JWT** للمصادقة
- **bcryptjs** لتشفير كلمات المرور
- **express-validator** للتحقق من البيانات
- **helmet** للأمان

## 📦 التثبيت والتشغيل

### المتطلبات الأساسية
- Node.js (الإصدار 16 أو أحدث)
- PostgreSQL (الإصدار 12 أو أحدث)
- npm أو yarn

### خطوات التثبيت

1. **استنساخ المشروع**
```bash
git clone <repository-url>
cd treasury-management-system
```

2. **تثبيت التبعيات**
```bash
npm run install-all
```

3. **إعداد قاعدة البيانات**
```bash
cd backend
npm run migrate
```

4. **تكوين متغيرات البيئة**
```bash
# في ملف backend/.env
DATABASE_URL=postgresql://username:password@localhost:5432/treasury_db
PORT=3001
JWT_SECRET=your-secret-key
FRONTEND_URL=http://localhost:3000

# في ملف frontend/.env
REACT_APP_API_URL=http://localhost:3001/api
```

5. **تشغيل النظام**
```bash
# تشغيل الخادم الخلفي والواجهة الأمامية معاً
npm run dev

# أو تشغيل كل منهما منفصلاً
npm run server  # الخادم الخلفي
npm run client  # الواجهة الأمامية
```

## 🚀 الاستخدام

### تسجيل الدخول
1. افتح المتصفح وانتقل إلى `http://localhost:3000`
2. سجل حساب جديد أو استخدم حساب موجود
3. بعد تسجيل الدخول، ستظهر لوحة التحكم الرئيسية

### إدارة الخزائن
1. انتقل إلى صفحة "الخزائن"
2. أضف خزائن جديدة أو عدل الموجودة
3. راقب الأرصدة والعمليات

### العمليات المالية
1. من لوحة التحكم، اختر الخزينة المطلوبة
2. اضغط على "إيداع" أو "سحب" أو "تحويل"
3. أدخل التفاصيل المطلوبة وأكد العملية

### التقارير
1. انتقل إلى صفحة "التقارير"
2. اختر نطاق التاريخ المطلوب
3. استعرض التقارير والرسوم البيانية

## 🔧 التطوير

### إضافة ميزات جديدة
1. أضف API endpoints في مجلد `backend/routes/`
2. أنشئ مكونات React في مجلد `frontend/src/components/`
3. أضف الصفحات في مجلد `frontend/src/pages/`
4. حدث التوجيه في `frontend/src/App.tsx`

### الاختبار
```bash
# اختبار الخادم الخلفي
cd backend
npm test

# اختبار الواجهة الأمامية
cd frontend
npm test
```

## 📝 API Documentation

### Authentication Endpoints
- `POST /api/auth/login` - تسجيل الدخول
- `POST /api/auth/register` - إنشاء حساب جديد
- `GET /api/auth/profile` - الحصول على بيانات المستخدم

### Treasury Endpoints
- `GET /api/treasuries` - جلب جميع الخزائن
- `POST /api/treasuries` - إنشاء خزينة جديدة
- `PUT /api/treasuries/:id` - تحديث خزينة
- `GET /api/treasuries/:id/balance` - جلب رصيد خزينة

### Transaction Endpoints
- `POST /api/transactions/deposit` - إيداع أموال
- `POST /api/transactions/withdraw` - سحب أموال
- `POST /api/transactions/transfer` - تحويل أموال
- `GET /api/transactions` - جلب المعاملات

### Reports Endpoints
- `GET /api/reports/summary` - التقرير المالي الشامل
- `GET /api/reports/treasury-performance` - أداء الخزائن
- `GET /api/reports/trends` - اتجاهات المعاملات
